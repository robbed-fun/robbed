# docs/services — Per-Service Designs

**Genre: DESIGN** ("what the system is") — one implementable design per service; code is a *transcription* of these docs, nothing more (development-flow.md §1/§2). **Owners:** `contracts.md` → robbed-contracts · `indexer.md` + `api.md` → robbed-indexer · `web.md` → robbed-frontend; cross-service interface changes are ratified per development-flow.md §6 (architect signs off). **Authority relation:** below the spec and CLAUDE.md, above code; each doc is authoritative for its own service's interfaces and internals. Does **not** belong here: task sequencing (→ `docs/plans/`), decisions (→ `docs/decisions.md`), how-to (→ `docs/runbooks/`). These filenames are load-bearing: agents' doc lists, doc-check named-doc resolution, and implementation-plan verify clauses cite `docs/services/*.md` — do not rename or move them.
