## Scope note

This plan covers the **API + WS legs of M2**: `M2-2` (OpenAPI), `M2-9` (read endpoints incl. §5.1 search — was orphan A-1, now in scope, P-6), `M2-10` (upload + metadata, §12.19), `M2-11` (moderation + SIWE admin + rate limits), `M2-12` (gate-7 metric hooks incl. the v1.2 funding-cluster share), plus the API/WS orchestration legs of `I-1`/`I-3`. The pure-indexer items (`M2-4…M2-8`, `M2-13/14`) live in [indexer.md](indexer.md); they are **cross-service dependencies here, not tasks**. In particular **`M2-8` is the sole authoring item for `apps/api/src/ws.ts`** (Bun WS fanout host) — it is owned by the indexer plan even though the file sits in this service's tree; this plan consumes it and adds no WS-fanout row (see the flagged ownership seam at the foot).

## Decide-it-yourself decisions

Implementation-approach calls I own (development-flow §3 / task framing): research the established pattern first, choose the boring correct option, record the basis in a code comment + here, prove with a test.

| Decision | ⇐ | Research source | Chosen approach (basis) | Proving test |
|---|---|---|---|---|
| **Rate-limit IP source behind CDN** | M2-11 | Hono docs (`getConnInfo`, header handling); Cloudflare `CF-Connecting-IP` docs | Trust a **configured** trusted-proxy header (`CF-Connecting-IP`, else rightmost `X-Forwarded-For` hop), **never** the raw client-settable leftmost XFF; header name is config (dev = socket IP). Basis: spoofable-XFF is the classic rate-limit bypass | unit: forged `X-Forwarded-For` does not evade the per-IP window; trusted header honored |
| **Cursor / pagination encoding** | M2-9 | PostgreSQL keyset-pagination docs; Hono query parsing | **Opaque base64url keyset cursor** encoding `(sort_key, id)` — not `OFFSET` (stable under inserts, O(1)); server-signed/validated so a malformed cursor 400s rather than scans | unit: round-trip encode/decode; tampered cursor → 400; keyset stable across an inserted row |
| **Error taxonomy** | M2-9 | Hono error-handling docs; api.md §2 envelope | **Closed enum of `error.code`** in `packages/shared` (`not_found`, `validation`, `rate_limited`, `oversized`, `unsupported_type`, `decode_failed`, `unauthorized`, `conflict`, `upstream_unavailable`) mapped to HTTP status by one central handler; no ad-hoc strings. Basis: consumers switch on a stable code, not prose. **Flag to hoodpad-shared** — the code enum is a cross-service shape (route through shared, don't redeclare) | unit: each thrown domain error maps to its code+status; unknown error → `500 upstream_unavailable`, never leaks internals |
| **SIWE session storage** | M2-11 | EIP-4361; viem `verifyMessage`/`verifySiweMessage`; Hono cookie middleware | **Stateless signed cookie** (HMAC over `{addr, iat, exp, nonce}`, 12h, HttpOnly/SameSite=strict) + **Redis-stored single-use nonce** (replay defense) + CSRF token on mutations. Basis: no session table to leak; nonce in Redis is the minimal server state; matches project key-custody trust model | unit: issue→verify→expire; replayed nonce rejected; expired/tampered cookie rejected; CSRF-missing mutation 403 |
| **Moderation queue worker tech (OI-A4)** | M2-11 | Redis list/stream docs; api.md §4.3 | **Minimal Redis-list worker** (not BullMQ) — one fewer dependency; visibility-timeout + backoff by hand; revisit if retry semantics outgrow it (recorded resolved-ish, implementation choice) | unit: job enqueue→dequeue→retry-on-throw with backoff; vendor-unavailable requeues |
| **Uncollected-fees RPC read placement** | M2-9 | api.md §3.4; viem contract read | Keep the `tokensOwed` read a **cold, 60s-cached read on the `/fees` route only** — never in the WS/publish hot path (protected by the row-9 import-graph test). Basis: api.md §3.4 assigns it to the API; the <500ms budget is a hot-path constraint, not a cold-read one | integration: `/fees` returns cached uncollected value; import-graph test proves no chain client in the fanout path |
