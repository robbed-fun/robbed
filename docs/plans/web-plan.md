## Architecture вҖ” Feature-Sliced Design (M3 restructure)

`apps/web` is organized with **Feature-Sliced Design** (https://feature-sliced.design) вҖ” see `docs/services/web.md` В§2.1 for the authoritative layout. Layers, strict downward imports only: `app вҶ’ views вҶ’ widgets вҶ’ features вҶ’ entities вҶ’ shared`; cross-slice access only via each slice's `index.ts` public API. The Next `app/` dir is routing-only (thin re-exports of a `views/*` screen); all code lives under `src/` (`src/app` = FSD app layer вҖ” providers + globals.css; `views` = the renamed FSD pages layer). Alias `@/*` вҶ’ `src/*`.

> **Note on the task table below:** rows M3-2/3/4/7 were authored and built against the earlier flat `components/`+`lib/` layout, then migrated to FSD in a pure no-behavior-change restructure (tsc/tests/build all green, same test count).
