# docs/security — Security Findings Registers

**Genre: LEDGER/REGISTER** (append-mostly) — consolidated adversarial-review findings with severities, owners, and dispositions; the milestone security gates' written record (spec §10). **Owner:** robbed-security (it refutes, it does not fix — findings route to the owning service agent; dispositions are recorded, never silently dropped). **Authority relation:** evidence, not authority — a finding feeds plan items and, when it embodies a decision, spec §12 via robbed-architect; the register itself never overrides spec or service docs. Does **not** belong here: threat *modeling* (→ `docs/threat-model.md`, a design-time artifact), fixes, or general review dockets (→ `docs/review/`). `findings-m1.md` is cited by implementation-plan verify clauses — path frozen.
