# docs/review — Verification & Review Dockets

**Genre: LEDGER/REGISTER** (append-mostly) — dated findings dockets from multi-agent verification passes and adversarial commit reviews; each is the disposition worklist for an architect ratification pass. **Owner:** robbed-architect (dockets are authored by the review session, dispositioned by the architect; per-finding fix owners are named inside). **Authority relation:** evidence, not authority — dispositions land in `docs/decisions.md` (and spec §12 when they embody a decision); a docket line never overrides spec or service docs. Does **not** belong here: contract-security gate registers (→ `docs/security/`) or open-item tracking (→ spec §13 / service-doc open-items tables). Convention: one file per pass, date-stamped (`findings-YYYY-MM-DD*.md`), never rewritten after disposition.
